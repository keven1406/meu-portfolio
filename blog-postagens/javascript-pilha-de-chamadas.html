JavaScript – Pilha de chamadas

Sabemos que um programa quando executado, tem inicio na primeira linha do código até a ultima linha. O fluxo de controle permite que o algoritmo seja lido da forma que deveria ser. Porém, vimos que existe declarações que nos permite modificar temporariamente esse fluxo, fazendo dar loops ou apenas pulando para uma linha de código se determinada condição for alcançada. Agora, vamos ver mais um modo de "modificar" a sequencia desse fluxo.

Pilha de Chamada (Call Stack)

As funções quando declaradas, nos permite executar um código dentro do nosso código principal sempre que ele for chamado. Mas ao fim da função, como o JavaScript volta para o fluxo ou contexto principal? Tudo volta ao normal porque o JavaScript monta uma pilha de fluxos ou contextos com todas as funções que estão sendo executadas no momento. Por exemplo:

function a ( ) { return "Eu quero café" } function b( ) { return a( ); } console.log (b( ));

Vamos prestar atenção no nosso exemplo acima. Ao iniciar o programa, o computador irá ler o console.log que está fazendo uma chamada para a função b(), notamos claramente que o bloco de códigos da função b() está retornando o texto da chamada da função a(), que vai fazer com que o programa execute os códigos contidos dentro dela. Dentro de a(), temos como retorno a string “Eu quero café!”, que será retornada onde foi feita sua chamada: a(), que está dentro da função b(). Retornado ao valor a(), o mesmo será retornado ao console.log que originalmente chamou a função b().

Entendendo como funciona o exemplo anterior, vamos entender como o programa sai do fluxo original que era o console.log, vai pra um novo fluxo, que é b(), vai para outro fluxo que é a(), volta para b(), e por fim continua no fluxo original em console.log.

Para fazer isso sem perder o fluxo original, o JavaScript abre um tipo de pilha que armazena o novo fluxo na parte de "cima". Seguindo o código que mostrei, a imagem abaixo irá representar como ficaria a pilha de contextos, ou melhor, pilha de chamadas:

Como podemos notar, o fluxo a() fica no topo da pilha por ser o ultimo a ser chamado , deixando os outros por ordem de chamada em baixo dele. Quando a função a() termina seu papel, ela é retirada do topo da pilha, e no seu lugar fica b(). Quando b() terminar o que deve fazer, ela é retirada da pilha sobrando apenas o fluxo original que vai dar continuidade ao programa.

Toda essa pilha fica armazenada na memoria do computador, para que ele possa se lembrar do contexto original do programa. Porém, quando a pilha fica muito extensa ao ponto de não termos mais memósria para armazena-la, acontece o seguite resultado:

out of stack space (Sem espaço na pilha) ou too much recursion (Muitas recursões)
