JavaScript – Escopo Léxico

Escopo léxico é como chamamos um conjunto de funções aninhadas. Um exemplo é aquelas bonecas russas chamada Matrioska. Ela tem a maior boneca iniciando primeiro e quando abrimos ela, temos uma sequencia de outras bonecas todas aninhadas uma dentro da outra como mostra a imagem abaixo: matrioskas.

Essas funções compartilham entre si as variáveis locais de suas antecessoras. Como assim? Primeiro, vamos explicar o que é variável local e variável global.

A variável local é um tipo de variável que só funciona dentro da função, deixando de existir quando a chamamos fora da função. Já a global pode ser usada tanto dentro de uma função quanto fora. Mas isso detalharei em outro post.

As variáveis locais só funcionam dentro da função onde ela está. No entanto, quando criamos uma função dentro de outra função (como a boneca Matrioska), a função principal, onde a nova função está contida, pode compartilhar com ela suas variáveis locais. Porem, a nova função não pode compartilhar suas variáveis locais com sua antecessora e nem com as funções vizinhas . Podendo apenas compartilhar se houver uma função contida dentro dela. Vamos ver um exemplo para tirar o nó do cérebro:

const x = function() { const result = 1; const somar = function(numero) { result += numero; } somar(3); return result; } console.log(x());

Resultado no console: 4

Como podemos notar, a function “x()” está compartilhando a variável “result” com a function “somar()”. No entanto não teria como ser ao contrário, já que somar() está contida dentro de x(). Logo seria impossível “somar()” compartilhar uma variável local com x().
