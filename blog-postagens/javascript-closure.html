JavaScript – Closure

Quando pensamos que variáveis locais são eliminadas assim que a função é terminada, ficamos bastante surpresos quando descobrimos que ainda podemos ter acesso a elas. Como? Por meio da closures. Vamos ver um exemplo bem simples:

function calc(n) { return function(somar) { return n + somar } }; const n = 2; const soma = calc(n); console.log (soma(8));

Resultado: 10

Muito interessante, não é? Pegamos o valor de “n”, que é 2, e colocamos na função “calc”. Essa função, por sua vez, chama a nova função que vai retornar o valor “n” mais a soma que iremos citar posteriormente. Note que “return funtion(somar)” não tem nome, só a declaração funcional e o paramento somar. Isso acontece por que essa função está sendo encaminhada para a variável “soma”. ou seja, o nome dela será “soma”, e o paramento “somar” será que vamos colocar na variável soma. Pela lógica ficaria assim:

soma = function(somar)

/* Aqui vemos a variável que foi transformada em função mais o parâmetro de valor 8.*/

Da para fazer inúmeras coisas, além do nome da função está bem resumido, precisando apenas chamar a variável soma e por um parâmetro para ela executar a função. O legal, é por que a função “calc” fica como se estivesse dentro da função “soma”. Logo todas as variáveis locais criadas em ‘calc” ficam salvas dentro de “soma”.
